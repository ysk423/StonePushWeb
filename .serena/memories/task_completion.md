# Task Completion Checklist

コード変更後、タスク完了とみなす前に`stone-push-web/`ディレクトリで実行:

1. `npx tsc --noEmit`（または`npm run build`で型チェック込みビルド確認）— このプロジェクトにlint/testスクリプトは無く、静的型チェックが主な自動検証手段
2. ブラウザでの実機動作確認（クリック操作含む目視）は開発環境上通常未実施（`Docs/spec.md`5章に既知の制約として明記）。UI/挙動に関わる変更をした場合はその旨をユーザーに伝える
3. `Docs/spec.md`（機能仕様）・`Docs/design.md`（実装アーキテクチャ）に影響する変更をした場合は、該当箇所を追随して更新する（このプロジェクトの運用方針。過去のコミットでも仕様変更とドキュメント更新をセットで行っている）
