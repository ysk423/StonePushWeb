# 戦績ランキングデータの削除手順（RECORD_DELETION.md）

> `records`コレクション（戦績ランキング、詳細は `LEADERBOARD.md`）に登録されたデータを削除する手順書。
> セキュリティルール（`firestore.rules`）でクライアントからの`update`/`delete`は禁止しているため、削除はFirebase Console操作か、管理者（あなた）のPCからのFirebase CLI操作でのみ行える。

---

## 0. 削除前に知っておくこと

- **削除は取り消せない**。Firestoreに標準のゴミ箱・復元機能は無い（Point-in-Timeリカバリという有料オプションはあるが、本プロジェクトでは未設定）
- 心配な場合は、削除対象のドキュメント内容をメモ・スクリーンショットしてから削除するとよい
- どの方法で削除しても、削除内容はランキング画面に**即座に反映される**（次回そのカテゴリを開いたときのFirestore再取得で消える）

---

## 1. 1件だけ削除する（Firebase Console／推奨・一番安全）

1. https://console.firebase.google.com/project/stone-push-web/firestore/databases/-default-/data を開く
2. `records` コレクションを開く
3. 消したいドキュメント（`playerName` や `category` の値で見分ける）をクリック
4. 右上の「⋮」（その他の操作）→「ドキュメントを削除」→ 確認ダイアログで削除

複数件を消したい場合も、コレクション一覧画面でチェックボックスから複数選択→削除ができる。

---

## 2. 1件だけ削除する（Firebase CLI）

ドキュメントIDが分かっている場合、`stone-push-web`ディレクトリで以下を実行：

```
npx firebase-tools firestore:delete records/<ドキュメントID> --force
```

- ドキュメントIDはConsoleの該当ドキュメント画面上部に表示されている文字列
- `--force`（`-f`）を付けないと確認プロンプトが出る。付けると即削除されるので、IDの指定ミスに注意

---

## 3. `records`コレクションを丸ごと削除する（要注意）

荒らしが酷い、あるいはテストデータを一掃したい場合など、**全カテゴリの戦績を全部消す**操作。`category`による絞り込みはできない（4章参照）。

```
npx firebase-tools firestore:delete records --recursive --force
```

- `--recursive`を付けないとコレクション直下のドキュメントしか消えない（今回はサブコレクションが無いので実質同じだが、明示的に付けておく）
- `--force`を外すと確認プロンプトが表示されるので、**初めて実行するときは`--force`を外して一度内容を確認してから**実行することを推奨

---

## 4. 特定カテゴリ（vs人間 / CPUよわい 等）だけ削除したい場合

CLIの`firestore:delete`はフィールド値での絞り込み削除に対応していない。以下のいずれかで対応する：

- **Console上で絞り込んで手動削除**：`records`コレクション画面で`category`フィールドの値が一致するドキュメントを目視で探し、複数選択して削除（件数が少なければこれで十分）
- **Admin SDKスクリプトを書く**（件数が多い場合。現状未整備、必要になったら別途用意する）：Node.jsスクリプトで`category == 'CPU_EASY'`のようなクエリを実行し、ヒットしたドキュメントを`batch.delete()`で一括削除する方式

---

## 5. 迷ったときの推奨フロー

1. ランキング画面やFirebase Consoleで問題のあるデータ（不適切な名前・明らかにおかしい手数など）を見つける
2. Consoleでそのドキュメントを開き、内容を確認（誤削除防止のため一応メモしておく）
3. 1件〜数件なら **1章（Console）** で削除。多数件なら **2章（CLI・IDを控えて1件ずつ）** か、カテゴリ全体の掃除なら **4章** を検討
4. 全部やり直したい・荒らしが酷いなど極端な場合のみ **3章（コレクション全削除）** を使う
