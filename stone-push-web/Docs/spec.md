## 2. ゲームルール（実装済み部分）

`GAME_RULES_AND_REQUIREMENTS.md` の 1章（ゲームルール）と完全に一致する内容を実装している。

- 4×4ボード×4枚（2×2配置）、左上・右下がDARK、右上・左下がLIGHT
- 初期配置：全4ボードで黒=row3全列（4個）、白=row0全列（4個）
- 先手は黒固定（Web版では設定変更UIは未実装。CPU対戦時は人間側が黒/白どちらを持つかのみ選択可能）
- 1ターン＝パッシブ移動（自分のホームボードで1〜2マス、8方向、経路上に石があれば不可）→ そのままの方向・歩数でアグレッシブ移動（パッシブと逆色のボードで、相手の石を1個まで押せる。2個連続で押すのは不可。押し出されたら消滅）
- デッドエンドルール：パッシブ移動の結果アグレッシブ移動が1手も無い場合、そのパッシブ移動自体を選択不可としてUIから除外
- 勝利条件：いずれか1枚のボードで相手の石を4個すべて押し出したら勝利

**Android版との差分は無し**（ルール自体はWeb版でも完全再現）。

## 3. 機能要件（実装済み／未実装）

| モード | 状態 |
|--------|------|
| vs CPU（やさしいのみ） | ✅ |
| vs 人間（パス＆プレイ） | ✅ |

### 3.2 スタート画面

実装済み：
- モード選択（vs 人間 / vs CPU）
- vs CPU 時の手番選択（先攻＝黒 / 後攻＝白）
- タイトル下に初期盤面の静的イラストを表示（実際の初期配置をそのまま流用、操作不可）

未実装（Android版にあるが今は無い）：
- 難易度選択（CPUは「やさしい」固定）
- 言語切替
- 設定・戦績画面へのリンク（ルール説明へのリンクは実装済み。詳細は3.11参照）

### 3.3 ゲーム画面

実装済み：
- 4枚のボードを2×2グリッドで表示
- 各ボードの右上相当の位置に残り石数（黒:N 白:N）をリアルタイム表示。いずれかの色が残り1個になったら、その数字を赤字・太字で強調表示（次に押し出されると敗北するため）
- 画面上部にフェーズインジケーター（現在のプレイヤー・フェーズテキスト表示）
- CPU思考中のテキスト表示（「CPU 思考中…」。盤面グリッドの下に固定表示することで、表示/非表示の切り替え時に盤面セルの位置がずれないようにしている）
- 後攻（白）でプレイ中は、自陣（白ホーム）が画面下側に来るよう盤面の表示順・各ボード内の行の描画順を反転（ゲーム内部の座標・ロジックは反転しない、表示のみの変換）

フェーズ別ビジュアルフィードバック（簡易版として実装）：

| フェーズ | Web版での表示 |
|---------|---------|
| PASSIVE_SELECT | 動かせる石を枠線でハイライト／動かせない自石は半透明 |
| PASSIVE_CONFIRM | 選択中の石をオレンジ枠でハイライト／移動先候補を枠線表示 |
| AGGRESSIVE_SELECT | 使用不可ボード（パッシブと同色）を暗転／動かせる石を枠線でハイライト |
| AGGRESSIVE_CONFIRM | 使用不可ボードを暗転／選択中の石をオレンジ枠でハイライト／移動先（常に1マス）を枠線表示 |

4枚のボードの中央交点には常時「ボーダー」ラベルを表示（ルール説明画面と同じ用語・同じ位置の視覚的対応）。「ボーダー」＝黒ホームと白ホームの上下の境目のみを太い線で強調し、左右のDARK/LIGHTの区切りは強調しない。

移動先の枠線表示は、押し出し対象（移動先に相手の石がある場合）でも石に隠れず常に見えるようにしている。

Android版にあるが未実装：
- PASSIVE_SELECT時の全移動先プレビュー（小ドット）— Web版は選択後にしか出さない
- 押し出しアニメーション、CPU移動アニメーション（すべて即時反映）
- エラーメッセージのカード表示（デッドエンドはそもそも選択肢に出ないため、選択エラー自体が原理上発生しない設計になっている）

### 3.5 フェーズ制御・操作フロー

Android版の仕様通りに実装済み：

```
PASSIVE_SELECT
  └─ 動かせる石をタップ ──────────────────► PASSIVE_CONFIRM
                                               │
                         同じ石を再タップ ◄──┤
                                               └─ 移動先をタップ ──► AGGRESSIVE_SELECT
                                                                        │
                            ハイライト以外をタップ／✕キャンセル ◄─────┤
                            （リードを巻き戻してPASSIVE_SELECTへ）     └─ 動かせる石をタップ ──► AGGRESSIVE_CONFIRM
                                                                                                    │
                                               同じ石を再タップ／✕キャンセル ◄─────────────────────┤
                                               （AGGRESSIVE_SELECTへ戻る）                          └─ 移動先（1マスのみ）をタップ ──► ターン確定
                                                                                                                                        └─ 勝利条件成立 ──► GAME_OVER
```

「フォロー」で動かす石をタップすると、方向・歩数はリードと同じに固定されているため移動先は自動的に1マスへ絞り込まれる（そのマスをタップして確定）。

- CPU手番（vs CPU）：人間側の操作はすべて無効化。CPUは500ms後にパッシブ・アグレッシブを連続適用（アニメーション無しで即座に反映）

### 3.6 CPU AI

| 難易度 | 状態 | アルゴリズム |

### 3.7 アンドゥ

未実装（拡張1）。

### 3.8 押し出しアニメーション

実装済み。フォロー（アグレッシブ移動）確定時、動かす石と押し出される石をその場でスライドさせ（`transform: translate`、約260ms）、盤外へ消える石はフェードアウトさせる。CPUの手番でも同様に再生される。アニメーション中は操作不可（`isAnimating`フラグでガード）。リード（パッシブ移動）自体のアニメーションは無し（即時反映）。

### 3.9 設定

未実装。効果音・BGM・言語のON/OFF切り替えやDataStore相当（Web版ではlocalStorage予定）への永続化は無い。

### 3.10 戦績

未実装。

### 3.11 ルール説明画面

実装済み（簡易版）。Android版の「5ページ横スワイプ式ページャー＋図解アニメーション」ではなく、1画面の縦スクロールテキスト（盤面構成・ターンの流れ・リード・フォロー・勝利条件の5トピックを見出しで区切って表示）。盤面構成のみ、DARK/LIGHTの色分けを再現した簡易図（静的、アニメーション無し）を表示。

アクセス経路：
- スタート画面の「ℹ ルール説明」ボタン
- ゲーム画面ツールバーの「ℹ ルール」ボタン（常時表示、ゲーム進行中でも開ける）

どちらから開いても「← 戻る」で元の画面（スタート画面 or 対局中のゲーム画面）にそのままの状態で戻る。

### 3.12 結果画面

実装済み：
- 勝者（黒/白）をオーバーレイ表示
- 「もう一度」ボタン（同じ設定でリスタート）
- 「メニューに戻る」ボタン

## 4. Android版から意図的に変更している点

| 項目 | Android版 | Web版 |
|------|-----------|-------|
| 永続化 | DataStore | localStorage（拡張3で対応予定、現状は永続化自体が無い） |
| UIフレームワーク | Jetpack Compose | フレームワーク無し（素のTypeScript + DOM操作、`innerHTML`による全体再描画方式） |
| DI | Hilt | 無し（モジュール関数を直接import） |
| アニメーション | 石の押し出し・CPU移動を段階的に演出 | 押し出し（フォロー確定時）のみスライド＋フェードアウトで演出。リード（パッシブ移動）自体は即時反映 |

## 5. 既知の制約・未対応ケース

- 合法なパッシブ移動が1つも無い（完全なデッドロック）状態になった場合の挙動は仕様として未定義（Android版・Web版とも）。CPU側は `chooseCpuTurn` が例外を投げる実装になっており、この状態に陥ると画面が固まる可能性がある。ルール上は稀なケースのため未対応のまま。
- ブラウザでの実機動作確認（クリック操作を含む目視確認）はClaude Codeの環境上未実施。`npx tsc --noEmit` と `npm run build` の静的チェックのみ通過確認済み。
- GitHub ActionsからのGitHub Pagesデプロイが `Branch "main" is not allowed to deploy to github-pages due to environment protection rules` により失敗する事象が発生中。リポジトリの Settings → Environments → github-pages の Deployment branches 設定で `main` を許可すれば解消する見込み（コード側の問題ではない）。
