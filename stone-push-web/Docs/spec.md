## 2. ゲームルール（実装済み部分）

`GAME_RULES_AND_REQUIREMENTS.md` の 1章（ゲームルール）と完全に一致する内容を実装している。

- 4×4ボード×4枚（2×2配置）、左上・右下がDARK、右上・左下がLIGHT
- 初期配置：全4ボードで黒=row3全列（4個）、白=row0全列（4個）
- 先手は黒固定（Web版では設定変更UIは未実装。CPU対戦時は人間側が黒/白どちらを持つかのみ選択可能）
- 1ターン＝パッシブ移動（自分のホームボードで1〜2マス、8方向、経路上に石があれば不可）→ そのままの方向・歩数でアグレッシブ移動（パッシブと逆色のボードで、相手の石を1個まで押せる。2個連続で押すのは不可。押し出されたら消滅）
- デッドエンドルール：パッシブ移動の結果アグレッシブ移動が1手も無い場合、そのパッシブ移動自体を選択不可としてUIから除外
- 勝利条件：いずれか1枚のボードで相手の石を4個すべて押し出したら勝利

**Android版との差分は無し**（ルール自体はWeb版でも完全再現）。

### 3.3 ゲーム画面

実装済み：
- 4枚のボードを2×2グリッドで表示
- 各ボードの右上相当の位置に残り石数（黒:N 白:N）をリアルタイム表示
- 画面上部にフェーズインジケーター（現在のプレイヤー・フェーズテキスト表示）
- CPU思考中のテキスト表示（「CPU 思考中…」。プログレスバーではなくテキストのみの簡易版）

フェーズ別ビジュアルフィードバック（簡易版として実装）：

| フェーズ | Web版での表示 |
|---------|---------|
| PASSIVE_SELECT | 動かせる石を枠線でハイライト／動かせない自石は半透明 |
| PASSIVE_CONFIRM | 選択中の石をオレンジ枠でハイライト／移動先候補をドット表示 |
| AGGRESSIVE_SELECT | 使用不可ボード（パッシブと同色）を暗転／動かせる石を枠線でハイライト／移動先候補をドット表示 |

Android版にあるが未実装：
- PASSIVE_SELECT時の全移動先プレビュー（小ドット）— Web版は選択後にしか出さない
- 押し出しアニメーション、CPU移動アニメーション（すべて即時反映）
- エラーメッセージのカード表示（デッドエンドはそもそも選択肢に出ないため、選択エラー自体が原理上発生しない設計になっている）

### 3.5 フェーズ制御・操作フロー

Android版の仕様通りに実装済み：

```
PASSIVE_SELECT
  └─ 動かせる石をタップ ──────────────────► PASSIVE_CONFIRM
                                               │
                         同じ石を再タップ ◄──┤
                                               └─ ハイライトセルをタップ ──► AGGRESSIVE_SELECT
                                                                               │
                            ハイライト以外をタップ／✕キャンセル ◄────────────┤
                            （パッシブを巻き戻してPASSIVE_SELECTへ）          └─ 移動先をタップ ──► ターン確定
                                                                                                       └─ 勝利条件成立 ──► GAME_OVER
```

- CPU手番（vs CPU）：人間側の操作はすべて無効化。CPUは500ms後にパッシブ・アグレッシブを連続適用（アニメーション無しで即座に反映）

## 4. Android版から意図的に変更している点

| 項目 | Android版 | Web版 |
|------|-----------|-------|
| 永続化 | DataStore | localStorage（拡張3で対応予定、現状は永続化自体が無い） |
| UIフレームワーク | Jetpack Compose | フレームワーク無し（素のTypeScript + DOM操作、`innerHTML`による全体再描画方式） |
| DI | Hilt | 無し（モジュール関数を直接import） |
| アニメーション | 石の押し出し・CPU移動を段階的に演出 | 無し（即時反映。拡張3で対応予定） |

## 5. 既知の制約・未対応ケース

- 合法なパッシブ移動が1つも無い（完全なデッドロック）状態になった場合の挙動は仕様として未定義（Android版・Web版とも）。CPU側は `chooseCpuTurn` が例外を投げる実装になっており、この状態に陥ると画面が固まる可能性がある。ルール上は稀なケースのため未対応のまま。
- ブラウザでの実機動作確認（クリック操作を含む目視確認）はClaude Codeの環境上未実施。`npx tsc --noEmit` と `npm run build` の静的チェックのみ通過確認済み。
